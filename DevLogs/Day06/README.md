\# Project Q 개발 기록 - 6일차



\## 1. 개발 목표



6일차의 목표는 기존 Pixel Shot 단일 카드 구조를 확장하여, 여러 카드를 보유하고 선택해서 사용할 수 있는 다중 카드 시스템을 구현하는 것이다.



기존에는 좌클릭 시 하나의 공격 카드만 사용했지만, 이번 작업에서는 Pixel Shot, Focus Shot, Wide Shot 3개의 카드를 만들고 플레이어가 상황에 따라 카드를 선택해서 사용할 수 있도록 구성하였다.



\---



\## 2. 구현한 기능



| 구분 | 구현 내용 |

|---|---|

| 카드 데이터 구조 | CardData 클래스로 카드 이름, 비용, 쿨타임, 피해량, 탄환 수 관리 |

| 카드 종류 관리 | CardType enum으로 카드 종류 구분 |

| 기본 카드 3종 | Pixel Shot, Focus Shot, Wide Shot 추가 |

| 카드 선택 | 1, 2, 3 숫자키로 카드 선택 |

| 카드 변경 | Q, E 키로 이전/다음 카드 선택 |

| 마우스 휠 선택 | 마우스 휠로 카드 변경 |

| 카드 사용 | 좌클릭으로 현재 선택한 카드 사용 |

| 카드별 MP 비용 | 카드마다 다른 MP 비용 적용 |

| 카드별 쿨타임 | 카드마다 다른 쿨타임 적용 |

| 카드별 탄환 방식 | 카드마다 탄환 수, 속도, 퍼짐 각도 다르게 적용 |

| 카드 UI | 현재 보유 카드와 선택 카드 표시 |

| 보상 연동 | 랜덤 보상으로 모든 카드의 데미지, 쿨타임, 탄환 속도 강화 가능 |

\## 6. 조작 방식



| 입력 | 기능 |

|---|---|

| 1 | Pixel Shot 선택 |

| 2 | Focus Shot 선택 |

| 3 | Wide Shot 선택 |

| Q | 이전 카드 선택 |

| E | 다음 카드 선택 |

| 마우스 휠 | 카드 변경 |

| 마우스 좌클릭 | 선택한 카드 사용 |
